# ADB over TCP

In this page, AoT denotes ADB over TCP and will be used interchangeably.

Table of Contents

  • Do I have to enable ADB over TCP everytime I restart?
  • Cannot enable USB debugging. What to do?
  • Can I block tracker or any other application components using ADB over TCP?
  • Which features can be used in ADB mode?

# Do I have to enable ADB over TCP everytime I restart?

Unfortunately, yes. But as of v2.5.13, you don't need to keep AoT enabled all the time as it now uses a server-client mechanism to interact with the system but you do have to keep the Developer options as well as USB debugging enabled. To do that, enable ADB over TCP and open App Manager. You should see working on ADB mode toast message in the bottom. If you see it, you can safely stop the server. For Lineage OS or its derivative OS, you can toggle AoT without any PC or Mac by simply toggling the ADB over network option located just below the USB debugging.

# Cannot enable USB debugging. What to do?

See enable USB debugging.

# Can I block tracker or any other application components using ADB over TCP?

Sadly, no. ADB has limited permissions and controlling application components is not one of them.

# Which features can be used in ADB mode?

Most of the features supported by ADB mode are enabled by default once ADB support is detected by AM. These include disable, force-stop, clear data, grant/revoke app ops and permissions. You can also install applications without any prompt and view running apps/processes.
